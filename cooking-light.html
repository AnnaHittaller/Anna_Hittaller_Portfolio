Project details

Cooking Light

Cooking Light is a full-stack web application that serves as a digital recipe collection, allowing users to upload, save, and explore recipes. The application provides a seamless experience for culinary enthusiasts to share their favorite recipes and to create a personalized collection. The app provides an optimal user experience across various devices and screen sizes thanks to a responsive design.

The project was first created collaboratively with Contentful as headless CMS, and was later modified individually to practice creating fullstack apps with routing endpoints and RESTful APIs with the help of Express server and Node.

Key features:

  • Recipe Management:

    CRUD operations: Users can create, read, update, and delete their recipes.

    Image upload: Integrated image uploading for recipe illustrations.

    Hero image carousel: The homepage displays a hero section with a dynamically rendered featured recipe carousel.

    Like functionality: Users can like the individual recipes, saving them into their own recipe book within the app, only available for logged-in users.

  • User Management:

    CRUD operations: The API is written so that users can register, update, and delete their profiles. In the present state of the project it can be tested with guest credentials, until all-round data safety measurements will be applied in the upcoming version.

    Local storage: User data is saved to local storage to enable saving recipes into their own recipe books.

  • MongoDB Database:

    Data storage: Utilized MongoDB to store user information, recipes, and associated data.

    Mongoose Schema: Designed data models using Mongoose for consistent and organized database interactions.

  • Express server:

    RESTful API: Built a RESTful API using Express to handle client-server communication.

    Endpoint routing: Structured API routes for user and recipe management.

Challenges & solutions:

  • Image upload: Efficiently handling image uploads for recipe illustrations is a must when building a cooking website. I decided to employ the Multer library to manage image uploads and storage.
  • Switching from headless CMS to own backend: Recreating the endpoints in Express to keep the same functionality and adding even more routes for image upload was challenging. I kept the existing frontend and rewrote the backend parts of the project to keep it running smooth.
  • Team communication & collaboration: Managing effective communication within a geographically distributed team. To overcome these difficulties, we implemented regular video meetings, Discord channels and project management tools to facilitate communication, track progress, and assign tasks. We also had to coordinate code changes and merges - to achieve this, we utilized Git and GitHub to manage codebase changes, review code, and resolve conflicts. Allocating tasks and ensuring each team member's contributions align with project goals was also not easy, but after assigning roles and responsibilities based on individual strengths and expertise we could effectively work together and deliver the project within deadline, and we were even able to add some extra functionalities (like hero image carousel and back-to-top button).
